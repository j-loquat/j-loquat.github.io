DevOps for agentic teams

QuestOps

DevOps gave us the factory line. QuestOps gives you the command room -- where you can see who did what, what passed, what changed, and what's safe to ship.

Agentic coding needs a command interface, not just chat. Games already solved real-time coordination: queues, minimaps, party management, fog-of-war, checkpoints. I'm calling this QuestOps: DevOps for agent swarms -- run your codebase like an RTS/RPG.

QuestOps Key Components
  • Clear Objectives (Quests)
  • Explicit Roles + Permissions (Parties)
  • Gated Stages (Quest Chains)
  • Evidence Trails (Chronicles)
  • Approvals + Rollbacks (Because Agents Make... Creative Choices)

RTS Mode

RTS already has the cockpit we need

When you can command fifty units under pressure, you can probably ship a login page. The UI we need already exists -- it just speaks the language of squads and queues.

  • Control Groups -> agent squads (Frontend / Backend / Testing / Research)
  • Build Queues -> task queues + dependencies (not "go do stuff")
  • Global Task Board -> real WIP limits, real flow
  • Event Log -> tests failed / tokens spiking / merge conflict
